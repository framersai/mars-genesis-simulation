# F23.2 — Corporate-quarterly smoke test (end-to-end F23 validation)

**Status:** design, awaiting approval
**Date:** 2026-04-23
**Scope:** one-off script that compiles + runs the `corporate-quarterly` scenario against a real LLM provider and asserts the post-F23 time-units contract on the returned artifact. Not a CI test (real LLM spend).

---

## Motivation

F23 renamed every `year` field in paracosm's public API to `time` and added `labels.timeUnitNoun` so scenarios can declare their own cadence. Typecheck + unit tests pass, but the rename is only truly validated once a non-year scenario runs end-to-end through `compileScenario()` + `runSimulation()` and produces an artifact where `trajectory.timeUnit.singular === 'quarter'`.

`scenarios/corporate-quarterly.json` ships with this release as the example non-year scenario (quarterly cadence over 3 years, 40 employees, 5 departments, growth-vs-discipline leader pair). This smoke is the validation that the scenario actually runs and the rename propagates correctly.

---

## Goals

1. Confirm `compileScenario()` accepts the quarterly JSON and generates hooks that reference `ctx.time` / `ctx.timeDelta` / `ctx.startTime` (not the legacy names).
2. Confirm `runSimulation()` emits events whose `data.time` / `data.timeDelta` are populated with quarterly values.
3. Confirm `RunArtifactSchema.parse()` succeeds on the produced artifact.
4. Confirm `artifact.trajectory.timeUnit` carries `'quarter'` / `'quarters'` through the full pipeline.
5. Confirm no legacy `year` / `startYear` / `currentYear` / `yearDelta` keys appear anywhere in the artifact.

Non-goals: dashboard rendering (Track A, F23.1), a pretty README showcase (deferred), full 12-turn runs, pair-divergence comparison.

---

## Flow

Script location: `scripts/smoke-corporate-quarterly.ts` (new).

Invocation: `npx tsx scripts/smoke-corporate-quarterly.ts`.

Steps:

1. **Read scenario.** Load `scenarios/corporate-quarterly.json` from disk, parse as `ScenarioPackage`-compatible JSON.
2. **Bust compile cache.** Delete any cached compiled hooks at `.paracosm/cache/*corporate-quarterly*` so the test exercises the compile path rather than reading a prior cached hook.
3. **Compile.** Call `compileScenario(worldJson, { provider: 'openai', model: 'gpt-5.4-nano', cache: true, cacheDir: '.paracosm/cache' })`. Expect success + hooks generated with post-F23 vocabulary.
4. **Run both leaders in parallel.** Pull the preset from the scenario's `presets[0].leaders` (Priya Kapoor + Marcus Reinhardt). For each leader:
   ```typescript
   runSimulation(leader, [], {
     scenario,
     maxTurns: 2,
     seed: 42,
     costPreset: 'economy',
     provider: 'openai',
   })
   ```
   Both wrapped in `Promise.all` so they share wall-clock time.
5. **Assert on each artifact:**
   - `RunArtifactSchema.parse(artifact)` returns without throwing.
   - `artifact.trajectory.timeUnit.singular === 'quarter'`.
   - `artifact.trajectory.timeUnit.plural === 'quarters'`.
   - `artifact.metadata.currentTime` is an integer strictly greater than the scenario's `setup.defaultStartTime` (1), matching the turn count times `defaultTimePerTurn` (1).
   - `artifact.finalState.metrics.population` exists and is a number (post-F23 `.metrics`, not the pre-F23 `.systems`).
   - Deep-scan the artifact JSON: no key named `year`, `startYear`, `currentYear`, `yearDelta`, `birthYear`, `deathYear` anywhere in the object tree.
6. **Persist outputs.** Write both artifacts to `output/smoke-corporate-quarterly-<leaderSlug>-<ISO8601>.json`.
7. **Log divergence.** For each turn, print `[<LeaderName>] Turn <n> @ Quarter <time>: <eventTitle> → <chosenDecision>` so the stdout trace reads like a narrative and shows the two leaders diverging.
8. **Exit status.** Non-zero on any assertion failure or parse error; zero on full success.

---

## Assertion helper

Deep-key scan uses a recursive walker over the parsed JSON. Finds any `key === 'year' | 'startYear' | 'currentYear' | 'yearDelta' | 'birthYear' | 'deathYear'` and throws with the JSON path for debugging. Implemented inline in the script (no new module).

---

## Cost envelope

- Compile: `gpt-5.4-nano` per hook × ~6 hooks ≈ $0.05-$0.10 one-time.
- Run × 2 leaders × 2 turns, `economy` preset: ≈ $0.10-$0.30 per leader = $0.20-$0.60 total.
- **Ceiling: $1.** Script aborts with an error message if `artifact.cost.totalUSD` for either leader exceeds $0.50.

---

## Success criteria

- `npx tsx scripts/smoke-corporate-quarterly.ts` exits 0.
- Both per-leader artifacts save to `output/`.
- All six assertions pass on both artifacts.
- Total observed spend ≤ $1.
- Stdout trace shows at least one event per turn per leader with the "Quarter N" label rendered correctly.

---

## Rollback

Delete the script + delete the output JSONs. No runtime code changes, no schema changes, no cache-version bumps. Fully additive.
